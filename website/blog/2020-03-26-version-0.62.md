Hoje estamos lançando a versão 0.62 do React Native, que inclui suporte para Flipper por padrão.

<!--alex ignore master-->

Esta versão chega em meio a uma pandemia global. Estamos lançando esta versão hoje para respeitar o trabalho de centenas de colaboradores que tornaram este lançamento possível e para evitar que a versão fique muito atrás do master. Por favor, esteja ciente da capacidade reduzida dos colaboradores para ajudar com problemas e esteja preparado para adiar a atualização, se necessário.

## Flipper por padrão

[Flipper](https://fbflipper.com/) é uma ferramenta de desenvolvedor para depuração de aplicativos móveis. É popular nas comunidades Android e iOS, e nesta versão habilitamos o suporte por padrão para aplicativos React Native novos e existentes.

![Screenshot of Flipper for React Native](/blog/assets/0.62-flipper.png)

O Flipper fornece os seguintes recursos prontos para uso:

- **Metro Actions**: Recarregue o aplicativo e acione o Dev Menu diretamente da barra de ferramentas.
- **Crash Reporter**: Visualize relatórios de crashes de dispositivos Android e iOS.
- **React DevTools**: Use a versão mais recente do React DevTools junto com todas as suas outras ferramentas.
- **Network Inspector**: Visualize todas as requisições de rede feitas pelas aplicações do dispositivo.
- **Metro and Device Logs**: Visualize, pesquise e filtre todos os logs tanto do Metro quanto do dispositivo.
- **Native Layout Inspector**: Visualize e edite o layout nativo gerado pelo renderizador do React Native.
- **Database and Preference Inspectors**: Visualize e edite os bancos de dados e preferências do dispositivo.

Além disso, como o Flipper é uma plataforma extensível, ele fornece um marketplace que puxa plugins do NPM para que você possa publicar e instalar plugins customizados específicos para seus fluxos de trabalho. Veja os plugins disponíveis [aqui](https://www.npmjs.com/search?q=flipper-plugin).

Para mais informações, confira a [documentação do Flipper](https://fbflipper.com/docs/features/react-native).

## Novos recursos de dark mode

Adicionamos um novo módulo `Appearance` para fornecer acesso às preferências de aparência do usuário, como seu esquema de cores preferido (claro ou escuro).

```js
const colorScheme = Appearance.getColorScheme();
if (colorScheme === 'dark') {
  // Use dark color scheme
}
```

Também adicionamos um hook para assinar atualizações de estado das preferências dos usuários:

```jsx
import {Text, useColorScheme} from 'react-native';

const MyComponent = () => {
  const colorScheme = useColorScheme();
  return <Text>useColorScheme(): {colorScheme}</Text>;
};
```

Veja a documentação para [Appearance](/docs/appearance) e [useColorScheme](/docs/usecolorscheme) para mais informações.

## Movendo Apple TV para react-native-tvos

Como parte do nosso [esforço Lean Core](/blog/#lean-core) e para alinhar o Apple TV com outras plataformas como React Native Windows e React Native macOS, começamos a remover o código específico do Apple TV do core.

Daqui em diante, o suporte do Apple TV para React Native será mantido em [react-native-community/react-native-tvos](https://github.com/react-native-community/react-native-tvos) junto com o pacote NPM `react-native-tvos` correspondente. Este é um fork completo do repositório principal, com apenas as mudanças necessárias para suportar Apple TV.

As versões do `react-native-tvos` serão baseadas na versão pública do React Native. Para esta versão 0.62 do `react-native`, por favor atualize os projetos Apple TV para usar `react-native-tvos` 0.62.

## Mais suporte para atualização

Quando a versão 0.61 foi lançada, a comunidade introduziu uma nova ferramenta [upgrade helper](https://react-native-community.github.io/upgrade-helper/) para apoiar desenvolvedores na atualização para novas versões do React Native. O upgrade helper fornece um diff das mudanças da versão em que você está para a versão que você está almejando, permitindo que você veja as mudanças que precisam ser feitas para sua atualização específica.

Mesmo com esta ferramenta, problemas surgem ao atualizar. Hoje estamos introduzindo mais suporte dedicado para atualização ao anunciar o [Upgrade-Support](https://github.com/react-native-community/upgrade-support). O Upgrade Support é um rastreador de issues do GitHub onde os usuários podem enviar problemas específicos à atualização de seus projetos para receber ajuda da comunidade.

Estamos sempre trabalhando para melhorar a experiência de atualização, e esperamos que essas ferramentas deem aos usuários o suporte de que precisam nos casos extremos que ainda não cobrimos.

## Outras melhorias

- **LogBox**: Estamos adicionando a nova experiência de erros e avisos do LogBox como opt-in; para habilitá-la, adicione `require('react-native').unstable_enableLogBox()` ao seu arquivo `index.js`.
- **React DevTools v4**: Esta mudança inclui uma atualização para o [React DevTools mais recente](https://reactjs.org/blog/2019/08/15/new-react-devtools.html) que oferece ganhos significativos de performance, uma experiência de navegação melhorada e suporte completo para React Hooks.
- **Melhorias de acessibilidade**: Fizemos melhorias na acessibilidade incluindo a adição de [accessibilityValue](https://reactnative.dev/docs/accessibility#accessibilityvalue-ios-android), props faltantes em [Touchables](https://github.com/facebook/react-native/commit/8c0c860e38f57e18296f689e47dfb4a54088c260), `onSlidingComplete` [eventos de acessibilidade](https://github.com/facebook/react-native/commit/c7aa6dc8270c0eabc913fe6c617c8131e3f4b3c5), e mudando o role padrão do componente Switch de `"button"` para `"switch"`.

## Breaking changes

- **Remover PropTypes**: Estamos removendo `propTypes` dos componentes do core para reduzir o impacto no tamanho do aplicativo do core do React Native e para favorecer sistemas de tipos estáticos que verificam em tempo de compilação em vez de em tempo de execução.
- **Remover accessibilityStates**: Nós [removemos](https://github.com/facebook/react-native/commit/7b35f427fd66cb0f36921b992095fe5b3c14d8b9) a propriedade obsoleta `accessibilityStates` em favor da nova prop `accessibilityState` que é uma maneira semanticamente mais rica para os componentes descreverem informações sobre seu estado para serviços de acessibilidade.
- **Mudanças no TextInput**: Removemos `onTextInput` [do TextInput](https://github.com/facebook/react-native/commit/3f7e0a2c9601fc186f25bfd794cd0008ac3983ab) pois é incomum, não está em conformidade com W3C, e é difícil de implementar no [Fabric](https://github.com/react-native-community/discussions-and-proposals/issues/4). Também removemos a prop não documentada `inputView`, e `selectionState`.

## Depreciações

- `AccessibilityInfo.fetch` já estava obsoleto, mas nesta versão nós [adicionamos um aviso](https://github.com/facebook/react-native/commit/523ab8333800afbfb169c6fd70ab6611fe07cc2a).
- Definir `useNativeDriver` agora é [obrigatório](https://github.com/facebook/react-native/commit/5876052615f4858ed5fc32fa3da9b64695974238) para suportar a mudança do padrão no futuro.
- O `ref` de um componente `Animated` agora é o componente interno e [depreciou](https://github.com/facebook/react-native/commit/66e72bb4e00aafbcb9f450ed5db261d98f99f82a) `getNode`.
