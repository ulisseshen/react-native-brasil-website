Estamos empolgados em anunciar o React Native 0.61, que inclui uma nova experiência de recarregamento que estamos chamando de Fast Refresh.

## Fast Refresh

Quando perguntamos à comunidade React Native sobre [problemas comuns](https://github.com/react-native-community/discussions-and-proposals/issues/64), uma das principais respostas foi que o recurso de "hot reloading" estava quebrado. Ele não funcionava de forma confiável para function components, frequentemente falhava ao atualizar a tela e não era resiliente a erros de digitação e erros. Ouvimos que a maioria das pessoas o desativava porque era muito pouco confiável.

No React Native 0.61, **estamos unificando os recursos existentes de "live reloading" (recarregar ao salvar) e "hot reloading" em um único novo recurso chamado "Fast Refresh"**. O Fast Refresh foi implementado do zero com os seguintes princípios:

- Fast Refresh **suporta completamente o React moderno**, incluindo function components e Hooks.
- Fast Refresh **se recupera graciosamente após erros de digitação** e outros erros, e retorna para um recarregamento completo quando necessário.
- Fast Refresh **não executa transformações de código invasivas**, portanto é confiável o suficiente para estar ativado por padrão.

Experimente e nos diga o que você acha! Se preferir, você pode desativá-lo no Dev Menu (Cmd+D no iOS, Cmd+M ou Ctrl+M no Android). Ativar e desativar é instantâneo, então você pode fazer isso a qualquer momento.

Aqui estão algumas dicas sobre Fast Refresh:

- Fast Refresh preserva o state local do React em function components (e Hooks!) por padrão.
- Se você precisar redefinir o state do React a cada edição, você pode adicionar um comentário especial `// @refresh reset` no arquivo com aquele componente.
- Fast Refresh sempre remonta class components sem preservar o state. Isso garante que funcione de forma confiável.
- Todos nós cometemos erros no código! Fast Refresh tenta renderizar novamente automaticamente depois que você salva um arquivo. Você não precisa recarregar o app manualmente depois de corrigir um erro de sintaxe ou um erro de runtime.
- Adicionar um `console.log` ou uma instrução `debugger` durante as edições é uma técnica de depuração interessante.

Por favor, reporte quaisquer problemas com Fast Refresh no GitHub, e nós iremos investigá-los.

## Outras Melhorias

- **Corrigido suporte a use_frameworks! do CocoaPods.** Na versão 0.60 fizemos algumas atualizações para integrar o CocoaPods por padrão. Infelizmente, isso quebrou builds usando [use_frameworks!](https://guides.cocoapods.org/syntax/podfile.html#use_frameworks_bang). Isso foi [corrigido na versão 0.61](https://github.com/facebook/react-native/pull/25619), tornando mais fácil integrar o React Native em seus projetos iOS que exigem build com frameworks dinâmicos.
- **Adicionado Hook useWindowDimensions.** Este novo Hook fornece automaticamente e se inscreve em atualizações de dimensões, e pode ser usado no lugar da API Dimensions na maioria dos casos.
- **React foi atualizado para 16.9.** Esta versão deprecia nomes antigos para os métodos de ciclo de vida UNSAFE\_, contém melhorias no `act`, e mais. Veja o [post do blog React 16.9](https://reactjs.org/blog/2019/08/08/react-v16.9.0.html) para um script de migração automatizado e mais informações.

## Breaking Changes

- **Remoção do React .xcodeproj.** Na versão 0.60, introduzimos suporte a auto-linking via CocoaPods. Também integramos o CocoaPods nas execuções de testes e2e, de modo que a partir de agora, temos uma maneira padrão unificada de integrar RN em apps iOS. Isso efetivamente deprecia o suporte ao React .xcodeproj, e o arquivo foi removido a partir da versão 0.61. Nota: se você já usa auto-linking da versão 0.60, você não deve ser afetado.
