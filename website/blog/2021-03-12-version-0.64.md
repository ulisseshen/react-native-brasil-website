Hoje estamos lançando o React Native 0.64 que inclui suporte para Hermes no iOS.

## Hermes opt-in no iOS

[Hermes](https://hermesengine.dev) é um motor JavaScript open source otimizado para executar React Native. Ele melhora o desempenho ao diminuir a utilização de memória, reduzir o tamanho do download e diminuir o tempo necessário para o aplicativo se tornar utilizável ou "tempo para interação" (TTI).

Com este lançamento, temos o prazer de anunciar que agora você pode usar o Hermes para desenvolver no iOS também. Para habilitar o Hermes no iOS, defina `hermes_enabled` como `true` no seu `Podfile` e execute `pod install`.

```ruby
use_react_native!(
   :path => config[:reactNativePath],
   # to enable hermes on iOS, change `false` to `true` and then install pods
   :hermes_enabled => true
)
```

Lembre-se de que o suporte do Hermes no iOS ainda está em estágio inicial. Estamos mantendo-o como opt-in enquanto executamos mais benchmarks. Encorajamos você a experimentá-lo em seus próprios aplicativos e nos dizer como está funcionando para você!

## Inline Requires habilitado por padrão

Inline Requires é uma opção de configuração do Metro que melhora o tempo de inicialização ao atrasar a execução de módulos JavaScript até que sejam usados, em vez de na inicialização.

Este recurso existe e tem sido recomendado há alguns anos como uma opção de configuração opt-in, listada na [seção de Performance de nossa documentação](/docs/performance). Agora estamos habilitando esta opção por padrão para novos aplicativos para ajudar as pessoas a terem aplicativos React Native rápidos sem configuração extra.

Inline Requires é uma transformação Babel que pega imports de módulos e os converte para serem inline. Como exemplo, Inline Requires transforma esta chamada de import de módulo de estar no topo do arquivo para onde é usado.

**Antes:**

```jsx
import {MyFunction} from 'my-module';

const MyComponent = props => {
  const result = MyFunction();

  return <Text>{result}</Text>;
};
```

**Depois:**

```jsx
const MyComponent = props => {
  const result = require('my-module').MyFunction();

  return <Text>{result}</Text>;
};
```

Mais informações sobre Inline Requires estão disponíveis na [documentação de Performance](https://reactnative.dev/docs/0.74/ram-bundles-inline-requires#inline-requires).

## Hermes com Suporte a Proxy

Adicionamos suporte a Proxy ao Hermes, habilitando compatibilidade com projetos populares da comunidade como react-native-firebase e mobx. Se você tem usado esses pacotes, agora pode migrar para o Hermes em seu projeto.

Planejamos tornar o Hermes o motor JavaScript padrão para Android em um próximo lançamento, então estamos trabalhando para resolver os problemas restantes que as pessoas têm ao usar o Hermes. Por favor, abra uma issue no [repositório GitHub do Hermes](https://github.com/facebook/hermes) se houver problemas restantes impedindo seu aplicativo de adotar o Hermes.

## React 17

O React 17 não inclui novos recursos voltados para desenvolvedores ou mudanças significativas que quebrem compatibilidade. Para aplicativos React Native, a mudança principal é uma [nova transformação JSX](https://reactjs.org/blog/2020/09/22/introducing-the-new-jsx-transform.html) permitindo que os arquivos não precisem mais importar React para poder usar JSX.

Mais informações sobre o React 17 estão disponíveis [no blog do React](https://reactjs.org/blog/2020/10/20/react-v17.html).

## Principais Mudanças de Versão de Dependências

- Removido suporte para Android API levels 16-20. O aplicativo do Facebook consistentemente remove suporte para versões do Android com uso suficientemente baixo. Como o aplicativo do Facebook não suporta mais essas versões e é a principal superfície de teste do React Native, o React Native também está removendo o suporte.
- Xcode 12 e CocoaPods 1.10 são obrigatórios
- Suporte mínimo do Node aumentado do Node 10 para Node 12
- Flipper atualizado para 0.75.1
