19 de maio de 2022 marca a 11ª celebração anual do Global Accessibility Awareness Day e queríamos atualizar todos sobre o progresso de acessibilidade que fizemos no React Native Framework. A Meta (anteriormente Facebook) foi a [primeira organização a assumir o compromisso GAAD em 2020](/blog/2021/03/08/GAAD-React-Native-Accessibility), comprometendo-se a tornar o framework React Native acessível.

> _"Esperamos que este compromisso facilite para os desenvolvedores que usam React Native criar aplicativos móveis totalmente acessíveis e inspire outras organizações a fazer compromissos semelhantes para um futuro mais acessível."_
>
> — [Mike Shebanek, chefe de acessibilidade da Meta, 2020](https://gaad.foundation/gaadpledge/)

O processo inicialmente começou com uma revisão completa e análise de lacunas do framework focada em como o React Native utilizava as APIs do iOS e Android para suportar recursos de acessibilidade. Dezenas de problemas foram corrigidos ou fechados desde então, cumprindo o compromisso de tornar o React Native acessível e avançando a acessibilidade do framework cada vez mais.

Não paramos por aí, e no início de 2022, revisamos e priorizamos os problemas restantes desta análise de lacunas no [Improved React Native Accessibility Board](https://github.com/facebook/react-native/projects/15) com base em seu impacto nos desenvolvedores e usuários finais.

## Correções que já chegaram em 2022

- [Android: Disabled state not announced/disabled functionality not applied for some components](https://github.com/facebook/react-native/issues/30840)
- [Android: Position in collection not supported by some components](https://github.com/facebook/react-native/issues/30977)
- [Android: Make links independently focusable by Talkback](https://github.com/facebook/react-native/pull/31757)

## Correções atualmente em andamento

- [iOS/Android: Text input error announcement](https://github.com/facebook/react-native/issues/30848)
- [Android: Form field error state announcement](https://github.com/facebook/react-native/issues/30859)
- [Android: Role description is announced before the components text, rather than after](https://github.com/facebook/react-native/issues/31042)

## Destaque do colaborador

<!-- alex disable he-she herself-himself her-him -->

Queremos reconhecer e enviar um enorme agradecimento a [Fabrizo Bertoglio](https://github.com/fabriziobertoglio1987) que contribuiu com várias correções de acessibilidade de alta qualidade para o React Native em 2021 e 2022.

Fabrizio tornou seu objetivo pessoal capacitar os usuários através de soluções de alta qualidade. Ao aprender a usar o TalkBack e VoiceOver ele mesmo, ele descobriu como pode ser difícil para usuários de leitores de tela experimentar aplicativos do dia a dia. Ele quer construir software que remova o atrito dessas experiências e agrade usuários com deficiência.

Seus pull requests têm sido de altíssima qualidade, bem documentados e incluem casos de teste completos. Muito bem, Fabrizio! A comunidade de Acessibilidade do React Native agradece suas muitas contribuições destacadas.

## O que vem a seguir em 2022

Nosso objetivo é corrigir o máximo possível dos problemas de acessibilidade restantes em 2022. Também estaremos revisando o backlog de [problemas reportados pela comunidade React Native](https://github.com/facebook/react-native/issues) para procurar quaisquer novas solicitações relacionadas à acessibilidade.
