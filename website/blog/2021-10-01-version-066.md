### Lidar com toques em views filhas fora dos limites do pai no Android

Graças a [@hsource](https://github.com/hsource) por adicionar suporte de interação para filhos renderizados fora dos limites da view pai via `overflow: visible`. Este é um caso de uso comum e alinha o React Native no Android mais próximo aos padrões web.

Encontre mais detalhes no [pull request](https://github.com/facebook/react-native/pull/29039).

### Novas Permissões Bluetooth no Android

Adicionamos suporte para [novas permissões Bluetooth](https://developer.android.com/about/versions/12/features/bluetooth-permissions) em preparação para o Android 12 e planejamos atualizar o `targetSDKVersion` para 31 no próximo lançamento.

### Melhor Suporte para Apple Silicon, Xcode 13 e iOS 15

Este lançamento fornece várias soluções para tornar as compilações do Xcode para iOS em máquinas Mac Apple Silicon (M1) mais confiáveis.

Notavelmente, o novo template de app agora inclui um workaround do CocoaPods (graças a [@mikehardy](https://github.com/MikeHardy)!).
Para aplicar, certifique-se de que o Podfile do seu app tenha `__apply_Xcode_12_5_M1_post_install_workaround(installer)` adicionado na etapa `post_install`.

Além disso, `RCT-Folly.podspec` foi [atualizado para prevenir falha do linker arm64](https://github.com/facebook/react-native/commit/8b6d7fddd65a9b5caf599e8ff7b090a176a6f11f).

Confira este [post](/blog/2021/09/01/preparing-your-app-for-iOS-15-and-android-12) que compartilhamos sobre preparar seu app para iOS 15 e Android 12.

### Hermes 0.9.0

O Hermes 0.9.0 é principalmente sobre fechar a lacuna entre o ponto de corte do lançamento do Hermes e este lançamento do React Native.

Entre ~400 commits, houve correções gerais de bugs ao lado de ganhos de memória e tamanho.

Veja [issue de lançamento do Hermes 0.9.0](https://github.com/facebook/hermes/issues/586) para mais detalhes

### Lançamentos Nightly e "Commitly"

Em um [post recente do blog](/blog/2021/08/19/h2-2021) compartilhamos que um de nossos objetivos no segundo semestre de 2021 é melhorar nosso processo de lançamento para ser mais rápido e mais estável. Como parte deste esforço, estamos trabalhando para tornar o React Native mais estável no main e para reduzir os bugs identificados durante nosso processo de Release Candidate.

Embora tenhamos publicado lançamentos nightly do React Native há mais de um ano, esses lançamentos não têm sido efetivamente usados pela maioria dos projetos. Eles agora são mais fáceis de acessar e esperamos usá-los como release candidates daqui para frente. Lançamentos nightly são publicados no npm sob a tag "nightly".

Para melhorar o processo de teste de commits individuais, o CI do React Native agora criará um artefato tarball para cada commit nos branches main e release, bem como para cada PR. Nós os chamamos de commitlies. Esses commitlies não serão publicados no npm, mas podem ser baixados diretamente do CircleCI. Veja as instruções abaixo.

Quer ajudar a mesclar um PR? Ao experimentar o commitly relacionado e verificar a mudança, você estará fornecendo um sinal valioso para ajudar a mudança a ser aceita!

#### Usando Lançamentos Nightly (Nightlies)

O processo para migrar seu projeto para um lançamento nightly do React Native é muito semelhante ao que você seguiria ao atualizar para uma versão regular, com a exceção de que ferramentas como o Upgrade Helper atualmente não funcionam com nightlies. Com isso em mente, recomendamos que você primeiro atualize seu projeto para o lançamento estável mais recente, se ainda não o fez. Em seguida, execute `yarn upgrade react-native@nightly` para instalar o lançamento nightly mais recente. Observe que pode haver mudanças adicionais necessárias para que seu projeto funcione corretamente em um lançamento nightly.

Assim como com um lançamento nightly, primeiro certifique-se de que seu projeto foi atualizado para a versão estável mais recente. Em seguida, vá para o [dashboard `react-native` no Circle CI](https://app.circleci.com/pipelines/github/facebook/react-native) e procure o workflow que foi acionado pelo commit em questão. Lá, você deve ver um job chamado `build_npm_package`. Esse job terá um painel "Artifacts" que fornecerá um link que você pode usar para baixar um arquivo tarball. Você pode então executar o seguinte:

```bash
# Update your react-native dependency to the tarball
# using your preferred package manager
$ yarn add <path to tarball>
$ npm add <path to tarball>
```
